# v0.2 Deferral Check

Date: 2026-04-30

## D-001 Dynamic Mid-Session Sandbox/Posture Switching

Capability: Dynamic mid-session sandbox/posture switching.
Target horizon: v0.4+ or earlier only with documented Codex support.
Deferred-to artifact: `CODEX_UPLIFT_ROADMAP_DEFERRAL_REGISTER.md` D-001.
Preserved seam: `PostureRecommendation` model with `launch_command`, `config_candidate`, `manual_app_steps`, and future `runtime_transition`.
What v0.2 must not do: Promise automatic runtime sandbox switching or silently launch full-access sessions.
Acceptance criteria for revival: Supported runtime transition API/event or verified client behavior.
Revisit trigger: Codex docs/client expose stable runtime posture transition.

## D-002 Exact Effective Config Resolution

Capability: Exact effective config resolution across every client/layer.
Target horizon: v0.3.
Deferred-to artifact: `CODEX_UPLIFT_ROADMAP_DEFERRAL_REGISTER.md` D-002.
Preserved seam: `observed`, `inferred`, `proposed`, `unknown`, `managed_or_overridden`, `requires_probe`.
What v0.2 must not do: Present a merged TOML as guaranteed effective state.
Acceptance criteria for revival: Doctor reports effective or explicitly unknown values across supported clients with version/trust annotations.
Revisit trigger: Reliable introspection/probe path appears.

## D-006 Full Plugin Publishing / Marketplace Lifecycle

Capability: Full plugin publishing and marketplace lifecycle.
Target horizon: v0.3.
Deferred-to artifact: `CODEX_UPLIFT_ROADMAP_DEFERRAL_REGISTER.md` D-006.
Preserved seam: Separate classic/plugin/hybrid install modes and machine-readable plugin component metadata.
What v0.2 must not do: Make hybrid duplicate skills the default or assume local plugin path handling equals publishable lifecycle.
Acceptance criteria for revival: Local marketplace validation, duplicate policy, update workflow, and external distribution docs pass.
Revisit trigger: Need to distribute beyond one local machine.

## D-008 Advanced Config Merge Engine

## D-009 Project Maturity Classifier

Capability: Project maturity classifier.

## D-015 Automatic Compaction Prompt Switching

Capability: Automatic compaction prompt switching.
Target horizon: v0.3/v0.4.
Deferred-to artifact: `CODEX_UPLIFT_ROADMAP_DEFERRAL_REGISTER.md` D-015.
Preserved seam: Prompt IDs, phase labels, profile metadata, and registry files.
What v0.2 must not do: Hard-code one prompt path or silently replace active compaction prompts.
Acceptance criteria for revival: Documented or probed prompt switching/application mechanism.
Revisit trigger: Evidence one prompt harms another workflow type.

## D-018 RTK Evaluation As Token-Efficiency Subject

Capability: RTK evaluation as a token-efficiency subject.
Target horizon: v0.2 evaluation protocol; v0.3 decision.
Deferred-to artifact: `CODEX_UPLIFT_RTK_EVALUATION_PROTOCOL.md`.
Preserved seam: Generic `OutputFilterProvider` adapter model.
What v0.2 must not do: Install RTK, run `rtk init`, add active RTK instructions, or assume transparent Codex command rewriting.
Acceptance criteria for revival: Evaluation shows material token savings without losing load-bearing detail, with raw-output recovery and no unsafe command-policy bypass.
Revisit trigger: User explicitly requests RTK evaluation or context-budget doctor flags shell-output bloat.

## D-020 Automatic Posture Routing
