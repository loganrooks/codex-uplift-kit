# v0.2 Scope Map

Date: 2026-04-30

## Scope Rule

v0.2 should become the first credible setup/posture assistant while preserving v0.3 seams. The smallest coherent slice must fix the two concrete installer defects from review, expose setup-assistant command seams, generate sensitive surfaces as candidates, and add manifest/status/uninstall safety basics.

## Capability Map

| Capability | v0.2 disposition | Notes |
|---|---|---|
| `inspect` basics | implement in this slice | Read-only inventory of Codex home, user `.agents`, package manifest, skills, agents, hooks, plugins, config presence, and uncertainties. |
| `config doctor` basics | implement in this slice | Report observed/inferred/proposed/unknown categories without claiming exact effective config. |
| `config candidate --profile <profile>` | implement in this slice | Generate candidate TOML for named posture profiles; do not merge active config. |
| Explicit install modes | implement in this slice | `classic`, `plugin`, `hybrid`, `minimal`; plugin mode skips standalone skills by default. |
| Component selection | implement in this slice | Support a conservative `--components` or equivalent model; preserve component IDs. |
| Plugin path validation | implement in this slice | Generate/validate marketplace metadata against chosen install location. |
| Duplicate skill handling | implement in this slice | Avoid duplicates by default; require explicit hybrid and report collisions. |
| Manifest | implement in this slice | Machine-readable package-owned file list with sha256 for installed/candidate files. |
| `status` | implement in this slice | Compare manifest entries against filesystem and detect missing/modified/unowned state. |
| `uninstall --manifest <path>` | implement in this slice | Remove only package-owned unmodified files; dry-run default or strongly supported. |
| `verify` | implement in this slice | Run non-destructive internal checks where possible; may be a lightweight command. |
| Hook candidates | candidate-only in this slice | Generate/sample only; never enable real hooks by default. |
| Rule candidates | candidate-only in this slice | Generate/sample only; rules are experimental command-policy surfaces. |
| Project inspect/candidate basics | candidate-only in this slice | Generate project observations and candidate files; do not overwrite active project config. |
| Compaction prompt candidates | candidate-only in this slice | Prompt catalog and candidate config only; no superiority claims. |
| RTK | test-only/probe-only in this slice | `rtk evaluate --plan-only` may generate evaluation plan; do not install RTK. |
| Dynamic posture switching | explicitly deferred | D-001. Preserve posture recommendation model. |
| Exact effective config resolver | explicitly deferred | D-002. Preserve observed/inferred/proposed/unknown model. |
| Advanced config merge engine | explicitly deferred | D-008. Generate candidates with merge metadata instead. |
| Project maturity classifier | explicitly deferred | D-009. Record project signals without over-classifying. |
| RTK integration | explicitly deferred | D-018/D-019. Preserve generic output-filter seam. |

## Non-Negotiable Safety Boundaries

- Do not mutate real user `~/.codex/config.toml`.
- Do not enable hooks, rules, full-access profiles, telemetry, or RTK in a live user config.
- Do not make `danger-full-access + auto_review` sound sandboxed.
- Do not allow plugin mode to silently duplicate standalone skill commands.
- Do not write marketplace metadata that cannot resolve to the copied plugin location.
- Do not uninstall unmodified/unowned files silently.
