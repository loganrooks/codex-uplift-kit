## UX Notes

- `--help` is clear enough for alpha, and the candidate-only safety note is
  visible in the main command surface.
- Plugin mode behavior is now understandable from command output: standalone
  skills are skipped by default, and marketplace fallback warns when temp Codex
  home sits outside temp user home.
- `compact candidate` communicates the important safety invariant directly:
  no active `config.toml`, `compact_prompt`, or
  `experimental_compact_prompt_file` activation.
- The npm `latest` dist-tag pointing at the alpha is potentially confusing, but
  acceptable while no stable version exists. Future release docs should keep
  alpha install commands explicit.

## v0.3 Inputs

- Add a live Codex plugin restart/install probe before stable messaging.
- Add config/profile validation against the actual local Codex runtime where
  observable.
- Improve post-install guidance so users know which generated candidates to
  review first and which should stay inactive.
- Consider a clearer `doctor`/`probe` split for v0.3 so observed runtime facts
  and recommendations do not blur together.
- Keep compaction prompt work evaluation-driven; alpha shipping confirms the
  candidate packaging path, not prompt superiority.
