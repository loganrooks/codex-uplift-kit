## Outcome

`codex-uplift-init artifact check --path <repo> --json` validates whether a
target project satisfies the universal artifact authority contract for its
selected control-plane level.

The check must be deterministic, fixture-tested, and able to distinguish hard
failures from warnings and advisory recommendations.

## User-Visible Result

Users can run one command to learn whether a project has a coherent planning
control plane for the level they selected or inferred. The output is useful to
humans and machine-readable for future agents.

Example:

```bash
codex-uplift-init artifact check --path . --level product --json
```

## Acceptance Criteria

| ID | Layer | Criterion | Verification |
| --- | --- | --- | --- |
| AC-1 | hard | CLI accepts `artifact check --path <repo> --json` and optional `--level <level>`. | `node --test` CLI parser tests |
| AC-2 | hard | JSON output contains `status`, `selectedLevel`, `findings`, and `summary`. | fixture snapshot tests |
| AC-3 | hard | Every finding contains `id`, `layer`, `severity`, `target`, `evidence`, and `remediation`. | schema assertion tests |
| AC-4 | hard | Exit code is `0` with no hard failures, `1` with hard failures, and `2` for invalid invocation or unreadable target. | process exit tests |
| AC-5 | hard | Checker distinguishes current, active work, scratch, evidence, and archive artifacts by path and metadata where available. | fixture tests |
| AC-6 | hard | Product/platform levels require current state, active work, governance, and artifact lifecycle docs. | product fixture test |
| AC-7 | hard | Minimal/research/software levels do not require full long-horizon roadmap/release docs unless selected by closure target or pack. | minimal and research fixture tests |
| AC-8 | hard | Scratch docs are never treated as authoritative current instruction. | scratch fixture test |
| AC-9 | warn | Missing optional templates or decisions directory produce warnings, not hard failures, unless selected level requires them. | warn-only fixture test |
| AC-10 | advisory | Subjective quality of roadmap, vision, or strategy is not a hard failure. | fixture with weak prose still exits `0` if hard gates pass |
